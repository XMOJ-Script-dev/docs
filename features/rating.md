# 用户评分

XMOJ-Script 内置了一套**用户评分系统**，用于直观展示用户在小明的OJ上的做题水平。

## 评分计算方式

> 注意：评分规则可能随版本迭代而调整。

用户评分基于**AC 率**（Accepted Rate）计算：

$$\text{评分} = \frac{\text{AC 提交数}}{\text{总提交数}} \times 1000$$

其中"总提交数"包含所有状态（AC、WA、TLE、MLE、RE 等，以及等待中、已忽略等情况）。

例如：某用户共提交 200 次，其中 AC 120 次，则评分为 $120 / 200 \times 1000 = 600$。

::: info 缓存说明
评分数据**在客户端缓存一天**，因此评分变化并非实时反映，通常在次日刷新。
:::

## 评分颜色对照

根据评分高低，用户名将以不同颜色显示：

| 颜色 | 评分范围 | 说明 |
|------|---------|------|
| 🔴 **红色** | ≥ 500 | AC 率 50% 以上 |
| 🟡 **黄色** | 400 ~ 499 | AC 率 40%~49% |
| 🟢 **绿色** | 300 ~ 399 | AC 率 30%~39% |
| 🔵 **蓝色** | < 300 | AC 率 30% 以下 |

## 评分在哪里显示？

- **讨论区**：帖子作者的用户名旁
- **排行榜**：用户列表中的用户名
- **短消息**：发件人/收件人用户名

## 与洛谷咕值的异同

| 对比维度 | XMOJ-Script 评分 | 洛谷咕值 |
|---------|-----------------|---------|
| 计算依据 | 纯 AC 率 | 活跃度 + 贡献 + 练习 |
| 实时性 | 每日缓存刷新 | 每日更新 |
| 社区贡献 | **不考虑** | 计入加分 |
| 主观因素 | 较少 | 较多 |
| 跨平台对比 | 仅限 XMOJ | 仅限洛谷 |

XMOJ-Script 评分**相对客观**，不受社区活跃度影响，更能反映用户的纯做题能力。
