# 获取测试点数据

::: warning 当前状态
该功能目前存在已知 Bug，使用时可能出现问题，正在修复中。
:::

## 功能介绍

当您的提交出现 **WA（Wrong Answer）**、**TLE（Time Limit Exceeded）** 或 **RE（Runtime Error）** 时，XMOJ-Script 的"获取测试点数据"功能可以帮助您直接查看该测试点的**输入数据**，从而加快调试速度，找出代码中的问题。

## 使用方法

1. 前往**状态页面**（提交记录列表）
2. 找到出现错误的那次提交，点击展开详情
3. 在测试点结果旁，点击"**获取数据**"按钮
4. 稍等片刻，输入数据将显示在弹出框中

## 注意事项

- 该功能仅对**有权限查看**的测试点有效，部分比赛在封榜期间可能限制查看
- 获取的数据仅为**输入数据**，不包含标准输出
- 数据量较大时加载可能较慢
- 如功能显示异常，请在讨论区[反馈问题](https://github.com/XMOJ-Script-dev/XMOJ-Script/issues)

## 调试建议

获取到测试点数据后，您可以：

1. 将输入数据复制到本地编辑器，保存为 `.txt` 文件
2. 在本地运行程序，重现错误
3. 使用调试器逐步追踪问题
4. 如有必要，可对比标程的输出（若题目提供）
